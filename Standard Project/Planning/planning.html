User Requirements

User requirement id format: UR[permissionIndex].[appSectionIndex].[requirementIndex]

Requirements for any user


Base of the app

-UR1.1.1: The user can login using his/her google account
-UR1.1.2: The user can login without having to enter their username and password (assuming they are logged in some app on the phone with a google account)
-UR1.1.3: The app will appear in the language of the userâ€™s phone, if this is either English or Dutch, it will default to English otherwise.
-UR1.1.4: The app should be installable and not contain any mistakes for any android version equal to or higher than 4.4
-UR1.1.5: The user should see news on the home screen of the app
-UR1.1.6: The user should be able to open a menu to navigate to specific modules
-UR1.1.7: The user should be able to see his/her own data and be able to alter it

Declarations module

-UR1.2.1: The user should be to see an overview of sent declarations
-UR1.2.2: The user should be able to see a seperate overview of accepted declarations
-UR1.2.3: The user should be able to search in declarations by name
-UR1.2.4: The user should be able to create a new declaration
-UR1.2.5: The user should be able to alter declarations that have been sent but not accepted
-UR1.2.6: The user should be able to enter product as well as travel expenses in declaration
-UR1.2.7: The user should be able to enter all required data about product expenses: description, date, had BTW, amount and BTW amount
-UR1.2.8: The user should not be able to use the BTW field if 'had BTW' is set to false, this should also have a visual hint
-UR1.2.9: The user should be able to take a picture of the receipt and attach it to the declaration
-UR1.2.10: The user should be able to attach a file of the receipt to the declaration
-UR1.2.11: The user should be able to enter all required data about the travel expenses: start city, end city, date and distance
-UR1.2.12: The user should be able to search declarations by price range
-UR1.2.13: The user should be able to search declarations by date range
-UR1.2.14: The user should be able to see his/her declined declarations at the top of the sent declarations list, with an indicator of it being declined

Additional requirements for logged in administrators


Declarations module

-UR2.2.1: The user should have a list of submitted declarations of others
-UR2.2.2: The user should be able to add remarks to the declarations of others
-UR2.2.3: The user should be able to approve and decline declarations

Extra Requirements

-UR1.2.13: The user should be able to search declarations by date range
-UR1.2.14: The user should be able to see his/her declined declarations at the top of the sent declarations list, with an indicator of it being declined
